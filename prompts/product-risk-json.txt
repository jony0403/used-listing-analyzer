  ],
  "chronicDefects": [
    { "title": "구체적 증상·부품·상태명", "detail": "구매 전 확인 이유와 실제로 어떤 문제가 생기는지", "level": "safe|caution|risk" }
  ],
  "verdict": "추천/주의/비추천 중 하나와 이유 한 문장"
}

규칙:
- 조사 메모에 나온 내용만 사용하세요.
- relatedIssues는 제품 관련 이슈·논란·주의점 또는 판매글 상태와 연결되는 제품군 기준 확인 포인트를 최대 3개로 정리합니다.
- chronicDefects는 반복 고장, 내구성 약점, 반복적으로 확인되는 구매 리스크를 최대 3개로 정리합니다. 정확한 모델 고질병이 뚜렷하지 않으면 제품군 기준으로 사진·본문에서 확인해야 할 구체 상태를 넣고, detail에 "제품군 기준" 또는 "정확한 모델 고질병은 뚜렷하지 않지만"처럼 범위를 밝힙니다.
- title에는 "최근 이슈", "고질병", "AS·유통" 같은 범주명 대신 구체적인 증상·부품·상태를 쓰세요.
- level은 구매 영향이 작으면 safe, 확인 필요면 caution, 수리비·사용성에 큰 영향이면 risk.
- 판매자에게 물어볼 질문은 만들지 마세요.
- "정보 부족", "검색했습니다", "AI 기능" 같은 메타 문구는 쓰지 마세요.
- 조사 메모가 "정확한 모델 고질병은 뚜렷하지 않음"이라고 판단해도, 판매글 상태와 연결되는 구체 확인 포인트가 있으면 caution 카드로 정리하세요.
- "공식 제품 불확실", "비공식 제품 가능성", "가짜 제품 가능성", "사기 가능성"은 카드로 만들지 마세요.
