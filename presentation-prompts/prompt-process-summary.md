# 프롬프트 설계 과정 정리

이 문서는 중고 매물 분석 도구의 발표 자료를 만들기 위해, AI 호출 과정 중 "유사 매물 검색 키워드 생성" 프롬프트를 어떤 문제의식으로 설계하고 개선했는지 정리한 발표용 메모다. 추후 HTML 기반 5페이지 슬라이드로 옮길 수 있도록 흐름 중심으로 작성했다.

## 0. 전체 AI 처리 흐름

프로젝트 전체에서 AI 호출은 한 번으로 끝나는 구조가 아니다. 확장프로그램이 매물 정보를 웹 분석 화면으로 전달한 뒤, 목적이 다른 AI 호출들이 순서대로 이어지는 구조다. 현재 문서에서 자세히 설명하는 검색어 프롬프트는 그중 1단계에 해당한다.

1. 제품 식별·키워드 추출·관련 시세 검색
   - 확장프로그램이 번개장터·당근마켓 매물 상세 페이지에서 제목, 본문, 가격, 판매자 정보, 이미지 URL을 수집해 분석 웹으로 보낸다.
   - AI가 사진·제목·본문을 보고 이 매물이 어떤 제품인지 식별한다.
   - 식별된 제품을 기반으로 동일·유사 매물을 찾기 위한 검색 키워드를 만든다.
   - 그 키워드로 관련 시세 매물을 수집한다.
   - 단, 부속품·세트 구성·구성품 누락 여부가 섞이면 시세 신뢰도가 떨어질 수 있으므로, 이 문제는 이후 별도 보정 단계로 다룰 예정이다.

2. 제품 고질병·관련 이슈 검색 및 사진 기반 하자 탐지
   - 식별된 제품명을 기반으로 해당 제품의 고질병, 자주 발생하는 하자, 구매 전 확인해야 할 이슈를 검색한다.
   - 판매자가 올린 사진을 AI가 분석해 찍힘, 기스, 파손, 구성품 누락 의심 등 시각적 하자를 찾는다.
   - 향후에는 찾아낸 하자를 좌표 기반으로 서버에 전달하고, 서버가 이미지 위에 직접 오버레이를 그려 표시하는 기능을 고려한다.
   - 이 단계는 단순 텍스트 요약이 아니라, 제품 지식과 이미지 비전 분석이 결합되는 단계다.

3. 시세·하자·판매자 정보 기반 가격 판단
   - 1단계에서 수집한 관련 시세와 2단계에서 찾은 제품 이슈·사진 하자·판매글 정보를 함께 사용한다.
   - AI가 이 매물이 적정가인지, 비싼지, 싼지, 사기 의심 정황이 있는지 판단한다.
   - 최종적으로 합당한 추천 가격 또는 가격 범위를 제시한다.
   - 이 단계는 단순 평균가 계산이 아니라, 상태와 리스크를 반영한 가격 판단 프롬프트가 필요하다.

4. 판매자에게 보낼 네고 대사 생성
   - 가격 판단 결과와 발견된 하자·시세 근거를 바탕으로 판매자에게 보낼 협상 문장을 만든다.
   - 무례하지 않으면서도 근거가 있는 네고 문구를 생성하는 것이 목표다.
   - 예를 들어 "시세보다 조금 높고, 사진상 사용감이 보여서 이 가격에 가능할까요?"처럼 사용자가 바로 보낼 수 있는 형태가 되어야 한다.

따라서 발표에서는 "AI가 한 번에 모든 판단을 한다"가 아니라, 제품 식별, 하자·이슈 분석, 가격 판단, 네고 문장 생성처럼 목적이 다른 AI 호출을 단계적으로 연결한 구조라고 설명하는 것이 적절하다.

## 1. 목표

이 문서에서 집중하는 기능의 목표는 사용자가 중고 플랫폼의 매물 상세 페이지를 열었을 때, AI가 사진·제목·본문을 함께 보고 같은 상품을 찾기 위한 가장 적절한 검색 키워드를 자동으로 도출하는 것이다.

단순히 판매자가 쓴 제목을 복사하는 것이 아니라, 실제 번개장터·당근마켓 검색창에 넣었을 때 동일하거나 유사한 매물이 잘 나오는 제품명·모델명·통칭을 만드는 것이 핵심이다.

## 2. 초기 문제

초기에는 AI가 판매글의 표현을 그대로 따라가거나, 검색어를 너무 짧게 잘라내는 문제가 있었다.

- `닌텐도 스위치2`가 `닌텐도 스`처럼 중간에서 잘림
- `반스 어센틱`이 `어센`처럼 불완전하게 줄어듦
- `새상품 같은 가방찾는다면~`, `단순개봉 플레이 판매`처럼 판매자의 홍보·상태 설명이 검색어에 섞임
- 후보 출력 과정에서 `json`, 코드펜스, 설명 문장이 버튼으로 노출됨
- 프롬프트가 특정 예시를 직접 들면, 모델이 그 예시를 일반 규칙처럼 오해할 위험이 생김

이 문제들은 모두 "제품 식별어"와 "판매글 맥락"을 구분하지 못할 때 발생했다.

## 3. 설계 방향

프롬프트는 특정 상품명이나 특정 금지어를 계속 추가하는 방식으로 만들지 않았다. 발표에서 강조할 핵심은 "하드코딩된 제품별 규칙이 아니라, 모든 매물에 적용되는 판단 절차"다.

현재 프롬프트의 중심 원칙은 다음과 같다.

- 사진, 제목, 본문을 함께 본다.
- 판매자 표기가 틀릴 수 있으므로 사진과 웹 검색으로 확인한 사실을 우선한다.
- 출력 전 내부적으로 단어를 두 부류로 나눈다.
- 제품 식별어: 브랜드, 라인, 모델, 세대, 품목
- 판매글 맥락: 상태, 거래조건, 사용 이력, 배송, 인증 가능 여부, 홍보 설명
- 최종 검색어에는 제품 식별어만 남긴다.

예를 들어 `닌텐도 스위치2 단순개봉 플레이 판매`는 판매 상태와 판매 행위를 제거하고 `닌텐도 스위치2`처럼 남겨야 한다. `지방시 안티고나 핸드백 (새상품 같은 가방찾는다면~)`은 권유 문장을 제거하고 `지방시 안티고나 핸드백`처럼 남겨야 한다.

## 4. 프롬프트 파일 분리

교수님께 프롬프트 과정을 보여주기 쉽도록 AI에게 전달하는 프롬프트는 `prompts` 폴더로 분리했다.

- `prompts/search-query-single.txt`: 최종 자동 수집에 사용할 단일 검색어 생성 프롬프트
- `prompts/search-query-candidates.txt`: 개발·검수 과정에서 최대 3개의 후보를 비교하기 위한 프롬프트

두 프롬프트는 같은 철학을 공유한다. 후보 생성은 사용자가 최종적으로 고르기 위한 기능이 아니라, 품질 확인과 자동 선택 로직 개선을 위한 중간 장치다. 최종 목표는 AI가 가장 적합한 키워드를 자동으로 선택하고, 그 키워드로 유사 매물을 수집하는 것이다.

## 5. 후처리 보강

프롬프트만으로는 모델 출력이 항상 안정적이지 않기 때문에 서버에서도 후처리를 적용했다.

- 코드펜스, JSON 키, 설명 문장 제거
- 단어 중간에서 잘린 검색어 보정
- 제목 전체를 그대로 복사하지 않도록 제한
- 상태·거래·사용 설명이 시작되는 지점 이후 제거
- 후보가 너무 길거나 판매글 문장처럼 보이면 버림

이 후처리는 프롬프트를 대체하는 것이 아니라, 모델 출력의 안전장치 역할을 한다.

## 6. 속도 최적화

정확도를 위해 이미지와 Google Search grounding을 사용했지만, 모든 요청에서 원본 이미지와 웹 검색을 무조건 사용하면 응답이 1분 가까이 느려질 수 있었다.

현재 구조는 다음처럼 최적화했다.

- 이미지는 최대 3장만 사용한다.
- 이미지 다운로드는 병렬로 처리한다.
- 이미지 1장당 타임아웃을 6초로 제한한다.
- 가능한 경우 400px급 이미지 URL로 낮춰서 전송한다.
- 먼저 빠른 멀티모달 응답을 시도한다.
- 결과가 부적절할 때만 Google Search grounding으로 재시도한다.

이렇게 해서 일반적인 매물은 빠르게 처리하고, 식별이 어려운 매물만 웹 검색을 사용하도록 했다.

## 7. 발표 슬라이드 초안

나중에 HTML 슬라이드로 만들 때는 다음 5페이지 구성이 적합하다.

1. 문제 정의: 중고 매물은 제목·사진·상태·시세가 불완전해 사람이 직접 판단하기 어렵다.
2. 전체 AI 파이프라인: 키워드·시세 검색, 하자·이슈 분석, 가격 판단, 네고 대사 생성.
3. 1단계 상세: 제품 식별어만 남기는 검색어 프롬프트 설계와 유사 매물 수집.
4. 확장 예정 단계: 제품 고질병 검색, 사진 하자 탐지, 좌표 기반 이미지 오버레이.
5. 최종 가치: 적정 가격 추천과 근거 있는 네고 문장까지 이어지는 구매 의사결정 보조.

## 8. 현재 결론

현재 프롬프트의 핵심은 "AI에게 특정 상품별 정답을 외우게 하는 것"이 아니라, 어떤 매물이 들어와도 제품 식별에 필요한 단어만 남기는 판단 절차를 주는 것이다. 이 방향이 발표에서도 가장 설득력 있는 설명 포인트다.

## 9. 제품 요약 프롬프트로 확장

검색어 생성만으로는 사용자가 "이 매물이 정확히 어떤 제품인지", "신품가는 어느 정도인지", "국내에서 누가 판매하는지"를 바로 판단하기 어렵다. 그래서 AI 처리 흐름을 단일 검색어 생성에서 제품 요약 단계까지 확장했다.

현재 제품 요약은 한 번의 AI 호출에 모든 것을 맡기지 않는다. 역할을 나눠 두 단계로 처리한다.

1. `prompts/product-identify.txt`
   - 판매글 제목, 본문, 사진을 보고 실제 제품명을 먼저 식별한다.
   - 신품가, 제조사, 설명을 쓰지 않고 제품 식별에만 집중한다.
   - 판매자의 상태·거래 조건·가격·홍보 문구는 제거한다.

2. `prompts/product-summary.txt` 및 서버 내부 제품 정보 조회 프롬프트
   - 1단계에서 식별한 제품명으로 Google 검색을 수행한다.
   - 한국 기준 신품가 또는 국내 정가/가격대, 제품 설명, 제조사·퍼블리셔·국내 판매처를 정리한다.
   - 같은 제품을 찾기 위한 검색어 후보도 함께 만든다.

이 구조는 "판매글을 요약하는 AI"가 아니라 "판매글에서 제품을 식별한 뒤, 제품 자체 정보를 다시 검색하는 AI"라는 점이 중요하다. 판매자의 설명이 부정확하거나 짧아도, 식별된 제품명으로 다시 검색하기 때문에 제품 정보의 안정성이 높아진다.

## 10. 검색어 품질 개선

실제 테스트에서 검색어에 `중고`, `가격`, 영문명 단독 후보가 섞이는 문제가 있었다. 예를 들어 `동키콩 바난자 가격`이나 `Donkey Kong Bananza Nintendo Switch 2`처럼 나오면, 번개·당근 검색어로는 너무 넓거나 한국 중고 플랫폼 검색 관점에서 어색하다.

이를 해결하기 위해 프롬프트와 서버 후처리를 함께 보강했다.

- 검색어에는 `중고`, `가격`, 상태, 지역, 거래 조건을 넣지 않도록 명시했다.
- 검색어는 한국어 표기를 우선하고, 영문명만 단독으로 쓰지 않도록 했다.
- 서버의 정규화 단계에서도 `중고`, `가격`을 제거하고, 한글이 없는 후보는 버린다.
- 여러 검색어가 한 줄에 섞여 나오면 쉼표·슬래시·줄바꿈 기준으로 분리해 각각의 검색 칩으로 보여준다.

발표에서는 이 부분을 "프롬프트 규칙만 믿지 않고, 서버 후처리로 한 번 더 안전장치를 둔 사례"로 설명할 수 있다.

## 11. 제품 이미지 검색 개선

초기 제품 이미지 기능은 검색 결과 첫 이미지를 그대로 보여주다 보니 이미지가 너무 작거나, 깨진 이미지가 뜨거나, 같은 프랜차이즈의 다른 시리즈 이미지가 뜨는 문제가 있었다.

개선 방향은 다음과 같다.

- 제품명과 검색어에서 핵심 단어를 뽑아 이미지 검색 후보를 점수화한다.
- 후보 이미지가 실제로 로드 가능한지 서버에서 먼저 검증한다.
- 외부 이미지 URL을 그대로 화면에 넣지 않고, 로컬 `/api/image-proxy`를 통해 보여준다.
- 분석 화면에서는 제품 이미지를 작은 썸네일이 아니라 대표 이미지 영역으로 크게 보여준다.
- 이미지 갱신은 임시 성격의 기능이므로 큰 버튼이 아니라 작은 새로고침 아이콘으로 낮춰 표시한다.

이 단계는 AI가 제품 정보를 찾는 것과 별도로, 사용자가 제품을 시각적으로 확인할 수 있게 돕는 보조 장치다.

## 12. 분석 화면 UX 피드백 반영

프롬프트 결과가 좋아도 화면에서 읽기 어렵거나 버튼 의미가 모호하면 사용성이 떨어진다. 그래서 실제 테스트 화면을 보면서 다음 UI 조정을 반복했다.

- 확장 버튼과 플로팅 버튼은 같은 동작으로 통일했다.
- 관련 매물은 페이지 로드 시 자동으로 뜨지 않고, 사용자가 버튼을 누를 때만 수집한다.
- `관련 매물 분석하기`는 의미가 딱딱해서 `비슷한 매물 찾기`로 바꿨다.
- `원본` 링크는 `판매글 열기`로 바꿔 클릭 후 이동할 위치를 더 명확히 했다.
- 본문·제품 설명은 말줄임만 하지 않고, 길 때만 내부 스크롤 영역을 만든다.
- 짧은 텍스트에는 스크롤 영역과 박스 스타일을 만들지 않아 카드 안에서 자연스럽게 보이도록 했다.
- 카드 안 요소가 테두리에 붙어 보이지 않도록 공통 패딩과 제품 요약 카드의 2열 레이아웃을 정리했다.
- 매물 사진 카드 제목은 `사진 3` 대신 `매물 사진`으로 바꿔 더 자연스럽게 했다.

발표에서는 이 과정을 "AI 프롬프트 설계와 화면 피드백이 같이 개선된 사례"로 정리할 수 있다. 모델 출력이 맞아도 사용자가 읽고 행동하기 쉬운 화면 구조가 필요하다는 점을 보여준다.

## 13. 현재 시연 흐름

현재 시연에서 보여줄 수 있는 흐름은 다음과 같다.

1. 사용자가 번개장터 또는 당근마켓 매물 상세 페이지에서 확장 버튼을 누른다.
2. 확장 프로그램이 제목, 본문, 가격, 판매자 정보, 사진 URL을 분석 웹으로 보낸다.
3. 분석 웹은 먼저 매물 기본 정보와 사진을 보여준다.
4. AI가 제품명을 식별하고, 식별된 제품명으로 제품 정보를 다시 검색한다.
5. 제품 요약 카드에 제품명, 국내 기준 신품가, 판매처/제조사, 제품 설명, 검색어 후보, 대표 이미지가 표시된다.
6. 사용자가 `비슷한 매물 찾기` 또는 검색어 칩을 누르면 번개·당근 검색 탭을 열고 비교 매물을 수집한다.

이 흐름은 "중고 매물 하나를 사람이 직접 검색하고 비교하는 과정"을 AI와 확장 프로그램이 단계별로 자동화한 것이다.
